# Visual Structure Decomposition

更新时间：2026-05-22

用途：把画面拆成可执行的导演事实，避免只写“日漫感、电影感、高级感”这类标签词。来源于 Super-i 提示词创作里“画面结构拆解法”的方法论沉淀。

## 核心判断

生成分镜图或 Seedance 全能参考前，先拆画面结构，再写风格词。

一个镜头至少拆成：

- 主体：谁是第一眼，谁是第二眼。
- 构图：主体在画面左/右/中，留白在哪里，前中后景怎么分层。
- 机位：平视、俯视、仰视、过肩、侧后方、低机位。
- 镜头距离：全景、中景、近景、特写、大特写。
- 光线：主光方向、阴影方向、天气/时间带来的环境光。
- 空间：角色、道具、门窗、桌面、道路、背景地标的相对位置。
- 时间状态：动作发生前、动作中、动作后，画面是哪个瞬间。

## 写法规则

不要让模型从抽象风格词里反推构图。先给它画面结构，再给风格。

推荐顺序：

1. 先写“画面第一眼是什么”。
2. 再写“角色/道具在画面里的位置关系”。
3. 再写“镜头怎么观察这个瞬间”。
4. 最后写风格、材质、光影和情绪。

错误写法：

```text
日漫电影感，青春感，氛围很好，镜头高级。
```

更稳写法：

```text
画面第一眼是画面左侧的短发少女半身侧脸，她站在雨后屋顶护栏前，右手握着旧磁带。
少年在画面右后方半步距离，只露出肩线和伸出的手。
镜头为中近景，略低于人物视线，背景城市天际线虚化，地面有雨水反光。
```

## 分镜图适配

分镜图不是最终彩色 keyframe。它应该把构图、屏幕方向、动作准备状态和镜头意图画清楚。

- 大主画面负责第一眼构图和空间关系。
- 小动作格只补充必要的手部、眼神、反应或环境动势。
- 不要把说明文字、时间码、表格和对白塞进图里。
- 当画面信息过多时，优先删掉风格词，不删主体关系和动作状态。

## QA 检查

生成前检查：

- 是否能一眼说出主画面第一眼。
- 人物/道具是否有明确左右、前后、远近关系。
- 当前画面是否是一个具体瞬间，而不是完整剧情摘要。
- 风格词是否压过了构图事实。

生成后检查：

- 画面是否像一个镜头，而不是海报或设定集。
- 角色站位是否能服务后续 Seedance 运动。
- 参考图有没有把原图构图错误地带进当前镜头。
