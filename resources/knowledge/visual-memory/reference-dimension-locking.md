# Reference Dimension Locking

更新时间：2026-05-22

用途：把参考图从“一堆图全塞进去”改成“每张图只锁一个维度”，减少 Image2 / Seedance 参考冲突和角色漂移。

## 核心判断

参考图不是最终画面目标，而是某个维度的锁。

每张图必须标注：

- 锁什么：角色身份、发型、服装轮廓、场景天气、空间布局、道具形状、光线质感。
- 不锁什么：原背景、原姿势、原构图、原镜头角度、无关道具、文字、水印。
- 冲突时谁优先：角色图管身份，场景图管天气和空间，分镜图管动作和构图，道具图管物件外形。

## 全能参考优先级

Seedance 全能参考推荐顺序：

1. 黑白分镜图：只管镜头顺序、构图、站位、动作方向、粗略切点。
2. 场景基准图：只管地点、天气、时间、空间氛围。
3. 角色参考图：只管身份、发型、服装轮廓、年龄感、体型。
4. 道具参考图：只管形状、比例、关键细节和手部交互。
5. 音频参考：只管对白节奏、表演节奏或声音气质。

如果分镜图里的人物和角色参考不一致，角色参考优先。分镜图里的临时人物只当占位。

## Image2 分镜图用图策略

Image2 做黑白分镜时，不要把所有参考图当成强约束。

- 可以带一张场景基准图稳定天气、空间、环境。
- 可以带锁定角色/道具图防止身份和关键物件漂移。
- 不要让风格图、成片帧、失败候选图成为上游权威。
- 不要把本地文件路径写进 prompt。

## 常见失败

- 角色参考图带着原背景，导致新镜头场景污染。
- 场景参考图压过分镜构图，角色站位错乱。
- 分镜图里角色画错，Seedance 把分镜临时人物当真实角色。
- 多张参考图同时管服装/身份/光影，模型混合出第三种人。

## QA 检查

- 每张参考图是否只有一个主要职责。
- 是否有冲突优先级。
- 分镜图是否只管导演意图，不管角色最终身份。
- 候选/失败/needs_review 图是否没有进入正向参考权威。
