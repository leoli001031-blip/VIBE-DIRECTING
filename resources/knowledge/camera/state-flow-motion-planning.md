# State Flow Motion Planning

更新时间：2026-05-22

用途：把动作从“动词清单”改成“状态流”，解决 Seedance / i2v 中人物动作平、乱、做不完的问题。

## 核心判断

视频 prompt 不要堆动词。一个镜头应该有一个核心动词，加上动作前状态、动作触发、动作后状态。

推荐结构：

```text
起始状态 -> 触发原因 -> 主动作 -> 微反应/结束状态
```

例如：

```text
少女低头握着旧磁带，手指已经收紧。
少年停在右后方，没有立刻说话。
少女因为听到脚步声抬眼，把磁带递到画面中央。
少年停半秒才接住，少女肩膀放松一点。
```

## 为什么比动词清单稳

动词清单会让模型同时执行多个动作：

```text
少女走过去、抬头、递出磁带、微笑、少年接住、两人对视。
```

状态流会让模型知道哪个动作是主动作，哪些只是前后状态：

```text
主动作是递出磁带。走近和对视不是独立事件，只作为递出前后的状态变化。
```

## 秒数和动作量

- 4-6 秒：一个主动作，最多一个微反应。
- 7-10 秒：一个主动作加一个反应，或两个紧密相连的动作节点。
- 11-15 秒：可以是一个小段落，但必须保持同一空间、同一轴线、同一目标。
- 快切例外：短时长也可多切点，但每个切点都必须是同一动作链上的简单状态。

## 分镜图适配

分镜大主画面应该画“动作即将发生”的状态，而不是动作完成后的结果。

- 递东西：主画面画递出前手的位置，动作格画递出方向。
- 推门：主画面画手压在门把上，动作格画门缝和反应。
- 跑动：主画面画重心压低准备冲出，动作格画方向线和对方反应。

## QA 检查

- 是否能指出唯一主动作。
- 是否有明确触发原因。
- 是否写了动作后微反应。
- 是否避免“然后、随后、接着”无限串联。
- 画面是否像起始状态，而不是已经演完。
