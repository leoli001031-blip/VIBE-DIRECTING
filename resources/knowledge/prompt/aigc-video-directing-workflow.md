# AIGC Video Directing Workflow

更新时间：2026-05-21

用途：把 Super-i 教程里对 Vibe Director 最有价值的 AIGC 视频经验沉淀成可注入规则。这个包只保留方法，不复制教程素材或大体积 assets。

## 核心判断

AIGC 视频生成不要从“一句完整大 prompt”直接开始。更稳的路径是：

1. 先把参考图、文本信息和视频目标拆成职责清楚的素材包。
2. 先做素材分析和视频策略判断，再生成分镜或时间线脚本。
3. 视频模型只负责无字画面、动作、镜头和视觉连续性；口播、字幕和品牌文字交给后期层。
4. 人物动作必须有行为动机和微反应，不要只写“站着、回头、微笑”。
5. 短视频按时间段拆成镜头单元，每段只承载一个视觉重点。

## 参考图职责分工

每张参考图必须先被标注职责，不能默认当成“成片目标”。

| 参考类型 | 只负责 | 不负责 |
|---|---|---|
| 人物图 | 五官、年龄感、发型、气质、穿搭倾向、身份可信度 | 原背景、原构图、最终场景 |
| 产品图 | 颜色、版型、材质、结构细节、卖点证据 | 人物姿势、场景氛围、广告文案 |
| 场景图 | 空间、灯光方向、生产/生活环境、可信氛围 | 人物身份、产品细节 |
| 风格图 | 光感、色调、镜头质感、画面密度 | 主体结构、故事动作 |

Prompt 编译时要显式写出职责，例如：

- `参考图1只用于老板人物形象，不使用其原始背景。`
- `参考图2只用于服装颜色、版型、纽扣、袖口、纹理等产品细节。`
- `参考图3作为主要工厂环境，人物需要自然融合到该环境中。`

如果文本和参考图在同一维度上重复或冲突，优先让文本定义结构，让参考图约束单一视觉维度。参考图的价值是减少随机性，不是替代文本把需求说清楚。

## 先素材分析再分镜

正式生成分镜前，先跑一轮只分析、不写脚本的素材策略。

素材分析至少回答：

- 人物适合承担什么角色：创始人、质检人、使用者、旁观者、被影响的人。
- 产品最适合被镜头证明的一个核心卖点是什么。
- 场景能提供什么信任背书或情绪背景。
- 文本信息里哪些卖点可以表达，哪些不能虚构。
- 这条视频的主目标是转化、信任、种草、情绪推进还是单一卖点强化。
- 哪些画面不该出现，避免模型补出不存在的资质、价格、销量、品牌字样。

第二步才把分析结果编译成 Story Flow / ShotSpec / Seedance 画面提示词。这样可以把“判断”和“生成”拆开，减少一上来就把参考图、卖点、口播、分镜全部混在一起的指令冲突。

## 无字画面分层

视频生成提示词默认只描述画面，不让视频模型生成文字。

必须分层：

- `画面提示词`：人物、产品、场景、动作、镜头运动、光线、构图、时长和画幅。
- `后期口播文案`：单独字段，供剪辑或 TTS 使用。
- `字幕/品牌文字`：后期图层，不进入 Seedance / i2v 画面 prompt。

硬性负约束：

- 无任何文字。
- 无字幕。
- 无标语。
- 无品牌字样。
- 无屏幕文字。
- 无海报文字。
- 口播只作为后期声音参考，不要求模型在画面中呈现文字。

## 动作动机和微反应

人物真实感来自“处境 -> 动机 -> 连续动作 -> 微反应”，不是来自脸部细节堆砌。

单人物动作公式：

```text
人物身份 + 具体场景 + 当前状态 + 动作原因 + 连续动作 + 微弱生命反应 + 镜头语言
```

可注入的微反应包括：

- 自然眨眼、轻微呼吸、肩膀或衣袖随呼吸起伏。
- 眼神短暂停顿、视线从物件移向画外。
- 手指收紧、松开、翻看、轻触材质。
- 嘴部自然开合、轻微手势，但不生成文字。
- 身体重心小幅转移，动作克制，不夸张表演。

多人物镜头里，至少区分一个人行动、另一个人反应。不要让所有角色同时平均运动。

## 时间线拆分

短视频要按时间线拆成可执行镜头块，不把多个意图压进一个长 prompt。

15 秒商业短片的默认结构可用：

| 时间段 | 视觉任务 | 写法重点 |
|---|---|---|
| 0-3 秒 | 钩子或身份建立 | 一个最强画面，不塞完整背景介绍 |
| 3-7 秒 | 产品证据 | 细节、材质、使用动作或卖点证明 |
| 7-11 秒 | 场景/信任背书 | 工厂、空间、环境、他人反应或流程证据 |
| 11-15 秒 | 收束和转化情绪 | 人物看向镜头、完成动作、留出口播位置 |

每个时间段都应包含：

- 画面内容。
- 镜头运动。
- 画面目的。
- 使用了哪些参考图及其职责。
- 无字画面提示词。
- 后期口播文案。

时间段不是越细越好。对 5-10 秒视频，优先保留一个核心动作和一个从属动作；复杂动作拆成多个 keyframe pair 或多个视频任务。

## QA 检查

生成前检查：

- 每张参考图是否只承担一个主要职责。
- 是否先有素材分析结论，再有分镜/时间线。
- 画面提示词是否剔除了字幕、品牌字、屏幕文字和后期口播。
- 人物是否有行为动机，而不是只有动作表面。
- 微反应是否克制且服务当前处境。
- 每个时间段是否只有一个视觉重点。

生成后检查：

- 人物是否被错误留在参考图原背景中。
- 产品细节是否被场景或人物参考污染。
- 场景是否出现未经提供的资质、销量、奖项、价格或品牌字样。
- 口播/字幕是否误入画面。
- 动作是否像状态流，而不是多个动词互相打架。
