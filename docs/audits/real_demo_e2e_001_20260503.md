# Real Demo E2E 001

This is the first small real-chain package for Vibe Core. It is image-first and Image2-first. It is not a Seedance/Jimeng/video test.

## Boundary

- `prepare` creates a file-first project package and waits for an imagegen subagent.
- `prepare` does not call Image2 and does not create or fake images.
- `verify` must fail while planned outputs are missing.
- Provider self-report alone never completes the run.
- A valid return path requires output file, provider observation sidecar, watcher event, manifest match, QA report, and preview plan.

## Commands

```sh
npm run real-demo-e2e-001:prepare
npm run real-demo-e2e-001:verify
```

The verify command exits non-zero until the Image2/imagegen subagent writes all planned outputs and sidecars.

## Project Package

Root:

```text
real-test-sandbox/real-demo-e2e/001
```

Prepared project facts:

- `project/project.vibe`
- `project/source_index.json`
- `project/visual_memory.json`
- `project/story_flow.json`
- `project/shot_layouts/*.json`

The scenario contains 8 shots, 2 scenes, 1 locked role, and 1 locked style. Only 2 real Image2 start frames are planned:

- `S01` start frame
- `S02` start frame

All other shots are queued or parked.

## Imagegen Subagent Packets

The imagegen subagent must read:

```text
real-test-sandbox/real-demo-e2e/001/task_packets/S01_start_frame_packet.md
real-test-sandbox/real-demo-e2e/001/task_packets/S02_start_frame_packet.md
```

Expected outputs:

```text
real-test-sandbox/real-demo-e2e/001/outputs/shots/S01/start.png
real-test-sandbox/real-demo-e2e/001/outputs/shots/S02/start.png
```

Required provider observation sidecars:

```text
real-test-sandbox/real-demo-e2e/001/provider_observations/S01_start_provider_observation.json
real-test-sandbox/real-demo-e2e/001/provider_observations/S02_start_provider_observation.json
```

The sidecar must be written by the imagegen subagent after it creates the image. The sidecar is evidence only; verify still blocks unless watcher, manifest, QA, and preview reports can be produced from the returned files.

## Reports

`verify` writes:

- `reports/watcher_events.json`
- `reports/manifest_match.json`
- `reports/qa_report.json`
- `reports/preview_plan.json`
- `reports/real_demo_e2e_report.json`

If outputs are missing, `real_demo_e2e_report.json` is written with `status: "blocked"` and the command exits with failure.
